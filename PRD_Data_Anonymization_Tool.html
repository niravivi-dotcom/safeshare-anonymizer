1. תקציר מנהלים

1.1 מה המוצר?

SafeShare הוא כלי desktop/web מקומי המאפשר למשתמשים עסקיים לא-טכניים להסיר מידע מזהה אישית (PII) מקבצי Excel, ולשתף אותם בבטחה עם ChatGPT, יועצים חיצוניים או אנשים שאינם מורשים לגשת לנתונים הרגישים המקוריים.

🎯 הערך המרכזי

"שתף נתונים עסקיים עם AI ואנשים - מבלי לחשוף סודות"

  • ✅ הכל מקומי - אפס נתונים יוצאים לאינטרנט בתהליך האנונימיזציה
  • ✅ פשוט לשימוש - Drag & Drop וקליק
  • ✅ בטוח מבחינה משפטית - תאימות GDPR וחוק הגנת הפרטיות
  • ✅ חכם - שומר על מבנה הנתונים לניתוח

1.2 למי זה מיועד?

  • מנהלי תיקים - רוצים לשתף נתונים עם ChatGPT לניתוח מבלי לחשוף ת.ז. ושמות
  • אנליסטים עסקיים - צריכים לשתף דוחות עם קונסולטנטים חיצוניים
  • צוותי משאבי אנוש - מכינים נתונים לדוגמה להדרכות
  • צוותי פיתוח - צריכים נתונים אמיתיים לבדיקות ללא חשיפת לקוחות

1.3 סיכום השקעה

פרמטר פירוט
זמן ל-MVP 2-3 שבועות (POC פונקציונלי)
תקציב ₪0 - טכנולוגיות open source בלבד
פלטפורמה Streamlit Web App (רץ מקומי)
רמת סיכון נמוכה - כל העיבוד מקומי, אין תלות בשרתים חיצוניים
יתרון תחרותי פשטות + מקומיות מוחלטת + תמיכה בעברית

2. הבעיה והזדמנות השוק

2.1 הבעיה העסקית

🚨 המצב הנוכחי

אנשים עסקיים רוצים לנצל AI וכלים מתקדמים, אבל נתקעים:

  1. חסימה משפטית - חוק הגנת הפרטיות (תיקון 13), GDPR, תקנות רגולטוריות אוסרים העלאת PII לשירותי ענן חיצוניים
  2. סיכון עסקי - חשיפת סודות מסחריים (שמות לקוחות, מעסיקים, נתונים פיננסיים) עלולה לגרום לנזק תחרותי ותביעות משפטיות
  3. תלות בטכנולוגיה - משתמשים עסקיים לא יודעים איך למחוק ידנית מידע רגיש מקבצים מורכבים
  4. אובדן הקשר - מחיקה ידנית הורסת את מבנה הנתונים ומונעת ניתוח משמעותי

2.2 תרחישים אמיתיים

תרחיש 1: מנהלת תיק רוצה עזרה מ-ChatGPT

מצב: יש לה קובץ Excel עם 500 שגיאות היזון חוזר. היא רוצה לשאול את ChatGPT: "תעדף לי את השגיאות הדחופות ביותר"

בעיה: הקובץ מכיל ת.ז., שמות, שמות חברות. אסור לה להעלות אותו.

תוצאה: היא תקועה - עושה הכל ידנית (4 שעות) במקום לקבל עזרה מ-AI (5 דקות)

תרחיש 2: צוות הדרכה צריך דוגמאות

מצב: יוצרים קורס הדרכה למנהלות תיקים חדשות. צריכים דוגמאות של קבצים אמיתיים.

בעיה: לא יכולים להשתמש בנתונים אמיתיים - חשיפת לקוחות.

תוצאה: יוצרים נתונים פיקטיביים באופן ידני (שעות עבודה) שלא משקפים מציאות

תרחיש 3: שיתוף פעולה עם יועץ חיצוני

מצב: יועץ עסקי מבקש דוחות לניתוח תהליכים.

בעיה: הדוחות מלאים במידע רגיש - לא ניתן לשתף.

תוצאה: היועץ מקבל תמצית ידנית חלקית, מפסיד תובנות חשובות

2.3 גודל השוק וההזדמנות

מיליוני משתמשי Excel בעולם
100M+ משתמשי ChatGPT פעילים
גבוה מאוד מודעות ל-GDPR ופרטיות

תובנה: השילוב בין רצון להשתמש ב-AI לבין חובות משפטיות יוצר ביקוש עצום לכלי אנונימיזציה פשוט ומקומי.

4.1 מה כן ב-MVP V1.0

תכונה פירוט
פורמט קלט Excel בלבד (.xlsx, .xls) - עד 10MB
שפות עברית + אנגלית
זיהוי PII ✓ ת.ז. (9 ספרות)
✓ שמות אנשים (NLP)
✓ כתובות מייל
✓ מספרי טלפון (ישראליים)
✓ כתובות (רחוב + עיר)
✓ מס' חשבון בנק (דפוס)
זיהוי אוטומטי המערכת סורקת ומזהה אוטומטית
עריכה ידנית משתמש יכול להוסיף/להסיר עמודות מרשימת הזיהוי
שיטת אנונימיזציה החלפה קונסיסטנטית (ת.ז. 123 → ID-001 תמיד)
פלט קובץ Excel אנונימי (אותו פורמט כמו קלט)

מחיקת מקור אוטומטית - הקובץ המקורי נמחק אחרי יצירת הקובץ האנונימי (עם אישור משתמש)
ממשק Streamlit web app פשוט - Drag & Drop
התקנה הרצה מקומית: pip install -r requirements.txt && streamlit run app.py

4.2 מה לא ב-MVP עתידי

תכונה למה נדחה לשלבים הבאים
Word, PDF Excel מכסה 80% מהמקרים. נוסיף בגרסה 2.0
קבצים >10MB MVP לא צריך לטפל בקבצים ענקיים. נייעל בהמשך
De-anonymization UI נשמור mapping file, אבל ממשק להחזרה יהיה בגרסה עתידית
ולידציה מתקדמת ניתוח אוטומטי שהאנונימיזציה הצליחה 100% - גרסה עתידית
Batch processing עיבוד מרובה קבצים בו זמנית - לא קריטי ל-POC
Audit log מפורט תיעוד מלא של פעולות - יתווסף כשנעבור לארגונים
Cloud version אם יהיה ביקוש, ניצור גרסת SaaS עם אבטחה מתקדמת
אינטגרציות API, חיבור לשירותים - לא נדרש ב-MVP

5. תכונות ויכולות מפורטות

5.1 זרימת UI - שלב אחר שלב

שלב 1: העלאת קובץ
  • אזור Drag & Drop גדול וברור
  • טקסט: "גרור קובץ Excel לכאן או לחץ לבחירה"
  • אינדיקטור התקדמות כשהקובץ נטען
  • הגבלה: רק .xlsx/.xls, עד 10MB
שלב 2: סריקה אוטומטית
  • מסך "סורק קובץ..." עם אנימציה
  • זיהוי אוטומטי של PII בכל העמודות
  • חישוב: כמה שדות רגישים נמצאו
  • זמן: 5-15 שניות (תלוי בגודל קובץ)
שלב 3: תצוגה מקדימה ואישור
  • כותרת: "מצאנו 347 שדות רגישים ב-5 עמודות"
  • טבלה: רשימת עמודות מזוהות + דוגמאות
  • לכל עמודה: checkbox (מסומן כברירת מחדל)
  • דוגמה: עמודה: "תעודת זהות" | דוגמה מקורית: 123456789 | אחרי: ID-001
  • כפתור: "הוסף עמודה באופן ידני" (למקרים שהמערכת פספסה)
שלב 4: הגדרות נוספות
  • ☑️ "שמור mapping file (מוצפן) - יאפשר שחזור בעתיד"
  • ☑️ "מחק קובץ מקורי אוטומטית אחרי אנונימיזציה"
  • אזהרה אדומה: "⚠️ קובץ המקור ימחק לצמיתות. ודא שיש לך גיבוי"
שלב 5: ביצוע אנונימיזציה
  • לחיצה על כפתור גדול: "🔒 אנונם עכשיו"
  • Progress bar: "מעבד שורה 230/500..."
  • זמן: 10-30 שניות
שלב 6: השלמה והורדה
  • הודעת הצלחה: "✅ הקובץ אונמם בהצלחה!"
  • סטטיסטיקה: "347 שדות הוחלפו | 0 שדות רגישים נותרו"
  • כפתורי הורדה:
    • 📥 הורד קובץ אנונימי (Excel)
    • 🔑 הורד mapping file (אם נבחר)
  • הצעה: "רוצה לעבד קובץ נוסף?"

5.2 אלגוריתם זיהוי PII

🆔 ת.ז. (תעודת זהות)

זיהוי: Regex: 9 ספרות רצופות

וולידציה: בדיקת ספרת ביקורת (אלגוריתם לוהן)

אנונימיזציה: ID-001, ID-002, ...

👤 שמות אנשים

זיהוי: Microsoft Presidio (NER model)

תמיכה: עברית + אנגלית

אנונימיזציה: PERSON-001, PERSON-002, ...

📧 אימייל

זיהוי: Regex: [\w\.-]+@[\w\.-]+\.\w+

אנונימיזציה: EMAIL-001@anon.com

📞 טלפון

זיהוי: Regex ישראלי: 05X-XXXXXXX, 0X-XXXXXXX

אנונימיזציה: PHONE-001, PHONE-002, ...

🏠 כתובות

זיהוי: Presidio LOCATION + רחוב/שדרות patterns

אנונימיזציה: ADDRESS-001

🏦 מס' חשבון בנק

זיהוי: Regex: דפוסי חשבונות ישראליים

אנונימיזציה: ACCOUNT-001

5.3 לוגיקת החלפה קונסיסטנטית

🧠 חכם: אותו ערך = אותה אנונימיזציה

דוגמה:

שורה ת.ז. מקורית אחרי אנונימיזציה
1 123456789 ID-001
2 987654321 ID-002
3 123456789 ID-001 ✓ (זהה לשורה 1!)
4 123456789 ID-001 ✓ (שוב זהה!)

למה זה חשוב? שמירת יחסים בין שורות - אם אותו לקוח מופיע 10 פעמים, הוא תמיד יקבל את אותו ID. ChatGPT יוכל לזהות דפוסים!

5.4 Mapping File (אופציונלי)

🔑 שמירת אפשרות שחזור

מה זה? קובץ JSON מוצפן שמכיל את המיפוי בין ערכים מקוריים לאנונימיים.

דוגמה (מוצפנת):

{
  "ID-001": "encrypted:AES256:a8f7d9b2c3e1...",
  "ID-002": "encrypted:AES256:x9k2m5n8p1q4...",
  "PERSON-001": "encrypted:AES256:z3v6b9m2k5..."
}

שימוש עתידי: אם המשתמש רוצה "לבטל אנונימיזציה" - יצטרך את הקובץ הזה + סיסמה.

אבטחה: הצפנת AES-256 עם סיסמה שהמשתמש בוחר.

7.1 מחסנית טכנולוגיות (MVP)

עיבוד Excel pandas + openpyxl קריאה/כתיבה של קבצי Excel

  1. לפני אנונימיזציה: וודא שיש לך גיבוי של הקובץ המקורי במקום בטוח
  2. סקירה ידנית: תמיד סקור את רשימת השדות המזוהים לפני אישור

  5. מחיקת מקור: רק אם אתה בטוח שיש לך גיבוי
